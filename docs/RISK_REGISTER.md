# Risk register

| Risk | Earliest signal | Mitigation | Convert into proof |
|---|---|---|---|
| Looks like document chat with extra screens | Main action is “ask” and outputs prose | Enforce Patch IR, deterministic projections/tests, and lifecycle navigation | Side-by-side naive-chat baseline |
| Too many GPT features dilute the story | Feature names dominate narration | Admit a capability only when it closes a concrete workflow gap | Ablation showing why that capability changes results |
| Medical-domain credibility gap | Realistic claims exceed synthetic evidence | Use fictional data, cautious language, software ground truth, human approval | Safety panel and blocked-state demo |
| Multi-agent is decorative | Agents repeat the same answer | Independent evidence scopes, separate findings, preserved dissent | Pediatric agent catches seeded exception others miss |
| Programmatic Tool Calling looks like a gimmick | Simple calls are wrapped in generated code | Reserve it for bounded parallel tests, joins, filtering, and aggregation | Trace thousands of raw assertions to few surfaced anomalies |
| Computer Use is brittle | Screen drift or timing breaks the demo | Own the fictional sandbox, stable accessibility labels, state recheck, reset | Show no-API staging plus safe stop before commit |
| Flutter Web visual ambition consumes build time | Pixel polish precedes a working vertical slice | Build signature journey first; tokenized components; golden QA | Codex render-inspect-refine log |
| Long context creates confident citation errors | Claim points to wrong or stale region | Immutable region IDs, checksum/version binding, provenance validation | Tampered/stale citation test blocks approval |
| Prompt caching leaks or goes stale | Incident evidence appears across run boundaries | Institution/run cache namespace, explicit breakpoints, invalidation tests | Cache-hit telemetry plus policy-update invalidation scenario |
| Deterministic tests merely encode the desired answer | Scenarios all mirror the demo | Hidden fixtures, baseline parity, ablations, immutable preregistration | Publish raw results and failed runs |
| Live API access or beta feature unavailable | Access probe fails | Probe early; isolate capabilities behind interfaces; label replays honestly | Capability matrix documents verified vs blocked |
| Deadline or registration failure | No Devpost registration/submission rehearsal | Register, rehearse submission, freeze early, keep checklist | Submission-readiness dashboard in repository |
| Absolute novelty claim is disproved | Adjacent vendor appears in search | Maintain dated competitor scan and narrow the integration claim | Judges see precise differentiation, not hype |
| Demo exceeds three minutes | Rough narration cannot fit | Time-box every beat and remove anything outside the transformation | Fast, legible story with one memorable reveal |

## Stop conditions

Do not add another capability until the current vertical slice can ingest the fixture, compile a typed patch, fail the seeded pathway, recompile, pass deterministic gates, and stop at approval. Do not record the final video until a clean reset-and-run succeeds repeatedly.
