# Hackathon victory matrix

Target category: **Work & Productivity**.

The four published judging criteria are weighted equally. Each claim below must be proved in the repository, the live product, or the sub-three-minute narrated video.

| Criterion | What judges should conclude | Proof we must ship | Failure to prevent |
|---|---|---|---|
| Technological Implementation | GPT-5.6 and Codex enable a reliable end-to-end institutional change loop that a simple chatbot cannot reproduce | Strict Patch IR; mixed-media evidence links; deterministic diff/test engine; bounded Programmatic Tool Calling; independent Multi-agent review; staged Computer Use; reproducible evals; Codex-authored tests and render QA | Feature parade, fake tool traces, model-only logic, unsafe autonomous write, undocumented fallback |

| Potential Impact | The product attacks a costly and globally credible coordination problem with a deployable human-accountable workflow | Evidence-backed shortage burden; before/after task map; time/error proxies; audit/rollback; US and Asian institutional relevance; clear path from synthetic sandbox to controlled pilot | Inflated market claims, unsupported lives-saved claims, Burkina-only framing, clinical-autonomy framing |
